# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository nature

This repo is the **published output** of a Hexo blog (https://elevenbeans.github.io), served as a GitHub user page from the `master` branch of `elevenbeans/elevenbeans.github.io`. It contains only generated static assets — there is no Hexo source tree, no `_config.yml`, no `package.json`, no `source/_posts/`, and no build/lint/test tooling in this repository.

Implications:
- **Do not treat this as a Hexo source repo.** Authoring new posts here means hand-editing generated HTML, which is almost never what the user wants. If the user asks to add/edit a post, confirm whether they intend to edit generated HTML directly or whether the Hexo source lives in a separate (private/local) repo that should be regenerated and re-deployed.
- There are no commands to build, lint, or test. Commits to `master` are the deploy — pushing publishes the site.
- Commit messages historically follow the auto-generated `Site updated: YYYY-MM-DD HH:MM:SS` pattern, indicating commits come from `hexo deploy` (which uses `hexo-deployer-git` to overwrite this branch).

## Layout (big picture)

- `index.html`, `archives/`, `categories/`, `tags/`, `page/` — generated index/archive/taxonomy pages.
- `YYYY/MM/DD/<slug>/index.html` — one directory per post, matching Hexo's default `permalink: :year/:month/:day/:title/`.
- `css/style.css`, `js/src/`, `assets/js/`, `lib/{jquery,fancybox}/` — theme assets from the **PolarBear** Hexo theme (a light theme based on Even). When the user references "the theme", this is it.
- `sitemap.xml`, `baidusitemap.xml` — generated by sitemap plugins; regenerated on each deploy.
- `elevenbeans.ico`, `about/` — favicon and the About page.

## Working in this repo

- A Hexo regeneration **overwrites everything** on this branch. Treat hand-edits to generated files as ephemeral unless the user confirms the source repo is unavailable.
- If you must edit generated HTML directly (e.g., a quick typo fix the user explicitly wants on the deployed site), keep the change minimal and warn the user it will be lost on next `hexo deploy` from source.
- The site is in Chinese (zh-CN) with mixed English; preserve language and tone when editing visible content.
